# vscode 配置 linux命令

通常来说, 在 vscode 中我们需要执行一些重复性比较多的一些命令，比如在打包的时候

npm run build
git add .
git commit -m"feat:new function"
git push origin
scp local_file remote_username@remote_ip:remote_folder

如果我们反复地敲这些命令的话会让我们的开发效率变得很低，所以我们需要通过某些途径来简化我们的步骤。

那么现在就想到了 .sh 文件 .sh是shell script格式的文件，在Linux环境下是可以直接运行的，所以我们只需要把我们的命令放到 .sh 文件下就可以每次快速执行。

但是执行. /test.sh 如果是在Windows环境下，在CMD窗口是无法直接执行，所以我们就需要去配置一下环境。 我们常见的 git 里面就有 bush, sh等的程序，所以我们主要把 git 的二进制文件配置到电脑的环境变量里面就可以了。

测试后发现我们 windows+R 发现 bush 后就可以编写我们的 linux 命令了，但是 vscode 还是不行。 出现的问题应该是我们没有以管理员身份运行 vscode, 所以我们右键选中 vscode 的属性，然后在兼容下选择以管理员模式运行即可。
