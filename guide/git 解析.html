# git 解析

1、本地关联远程分支

  • 已有分支关联远程分支
git branch --set-upstream-to=origin/remote_branch local_branch

tips: The local_branch must exist first, or it throw some errors.
(也就是上面这行命令的 locak_branch 需要是已经建立好的)
=> the error just like ... doesn't not exist
  • 基于上面的关联我们可以用以下的提交方式
git push origin local_branch:remote_branch

2、当出现 fatal: refusing to merge unrelated histories 的时候

  • git pull 尝试过会出现这个问题 => 有可能是因为不同分支之间的历史记录不一样

解决的办法： --allow-unrelated-histories

git pull --allow-unrelated-histories

# checkout 操作

    `git checkout -h` 查看命令具体添加的一些参数（其他一些命令也类似）

    -b <branch>           create and checkout a new branch
    -B <branch>           create/reset and checkout a branch
    -l                    create reflog for new branch
    --guess               second guess 'git checkout <no-such-branch>' (default)
    --overlay             use overlay mode (default)
    -q, --quiet           suppress progress reporting
    --recurse-submodules[=<checkout>]
                          control recursive updating of submodules
    --progress            force progress reporting
    -m, --merge           perform a 3-way merge with the new branch
    --conflict <style>    conflict style (merge or diff3)
    -d, --detach          detach HEAD at named commit
    -t, --track           set upstream info for new branch
    -f, --force           force checkout (throw away local modifications)
    --orphan <new-branch>
                          new unparented branch
    --overwrite-ignore    update ignored files (default)
    --ignore-other-worktrees
                          do not check if another worktree is holding the given ref
    -2, --ours            checkout our version for unmerged files
    -3, --theirs          checkout their version for unmerged files
    -p, --patch           select hunks interactively
    --ignore-skip-worktree-bits
                          do not limit pathspecs to sparse entries only
    --pathspec-from-file <file>
                          read pathspec from file
    --pathspec-file-nul   with --pathspec-from-file, pathspec elements are separated with NUL character

# git log

查看 git 历史的一些版本号用于回退版本

git remote set-url origin <remote_url>  修改指定的远程仓库
git remote add origin <remote_url>  添加远程仓库
git pull origin <remote_branch>  拉取指定分支的代码
git reset --hard [版本号] -> 回退到指定的版本号
